# 双通道机制：信念校准与偏好校准如何运作

> 前置阅读：[理论基础](01-theoretical-foundations.md) 解释了两个通道*为什么*必须分离。本文解释它们*如何*工作。

## 根本问题：一条不可消除的鸿沟

搜参是离线过程：固定一批历史日志，采样数千组参数，评估每组的影响力分布，选出最优解。前提假设是离线数据能代表线上现实。

**这个假设永远不完全成立。** 离线数据是历史快照，线上流量是活的——用户行为在变、竞品在变、季节性在变。一组离线搜出来 I_gmv 提升 +12% 的参数，上线后可能只涨 3%，甚至反跌。

如果搜参是"在地图上规划路线"，离线-线上 gap 就是"地图跟实际地形不完全一致"。传统做法依赖工程师手动判断和调整——慢、不可积累、且经验留在人脑中无法沉淀为系统状态。

Sortify 通过两个平行修正通道将这个反馈回路自动化。

## 信念通道：修正"离线说的话可以信几分"

### 线性迁移模型

系统为每对离线-线上指标维护一条线性映射：

```
线上 uplift ≈ slope × 离线 uplift + intercept
```

例如 `GMV~I_gmv` 关系：slope=0.6, intercept=-0.05，含义是"离线 I_gmv 每提升 1%，预期线上 GMV 提升约 0.6%，再减去 5% 的系统性折扣。"

### 为什么主要调截距而非斜率？

这里有一个关键假设：离线与线上指标的**边际关系**短期内相对稳定（斜率慢变），但**绝对水平**频繁漂移（截距快变）。

这在实践中是合理的。如果今天离线 I_gmv 提升 1% 对应线上 GMV 提升 0.6%，那提升 2% 大约对应 1.2%——比例关系短期不会剧变。但基线水平会因外部环境（大促、季节、竞争格局）持续漂移。

类比温度计：如果它系统性地偏高 2 度，每个读数都偏高相同的常数。实际温度变化之间的关系是保持的——刻度因子是对的，但零点偏了。修正截距（零点）就能修复偏差而不扰动刻度因子。

### LMS：稳定之手

LMS（最小均方）算法每轮更新斜率和截距：

```
误差 = 观测线上值 - 预测线上值
slope     += eta × 误差 × 离线值
intercept += eta × 误差
```

学习率 eta=0.2，即每轮只修正 20% 的残差。

特性：
- **无条件**：每轮都更新，不管观测是信号还是噪声
- **固定步长**：不区分大的结构性变化和小的随机波动
- **收敛但慢**：如果截距需要移动 0.06，大约需要 15 轮（超过两天）才能收敛到 95%

### LLM：选择性跳变

当底层分布突变（如大促结束、竞品上线）时，LMS 太慢。LLM 元控制器可以在单轮内做最大 ±0.1 的截距跳变，将原本需要 10+ 轮的收敛压缩到 1-2 轮。

关键的是，LLM 也可以选择不动。当数据噪声大或证据模糊时，它返回 `delta_intercept = 0` 并给出理由。在高方差场景下，不动往往是最优策略。

两者形成**双速率系统**：

| | LMS | LLM |
|---|---|---|
| 步长 | eta × 误差（小步，约 0.01） | delta_intercept（大步，最大 ±0.1） |
| 频率 | 每轮无条件 | 每轮可调，也可选择不调 |
| 输入 | 当前 1 个观测对 | 最近 20 轮 + 30 条更新记录 |
| 能力 | 梯度下降，无过滤 | 模式识别，选择性干预 |
| 类比 | 自动巡航 | 飞行员干预 |

每轮内两者的效果是加法叠加的：`最终截距 = 初始值 + LMS 修正 + LLM 修正`。

## 偏好通道：修正"这条红线该绷多紧"

### 惩罚权重作为拉格朗日乘子

每个约束有一个惩罚权重 lambda（类似拉格朗日乘子）。例如若某条约束的 lambda 取 10^4 量级，则每超出允许范围 1%，目标分会扣掉 `lambda × 0.01²` 这一量级的分数。

### 乘性更新

系统对 lambda 做乘性更新：

```
lambda_new = lambda_old × exp(step)
```

- 约束被线上数据违反：step > 0 → lambda 增大 → 下一轮搜索更重地惩罚违约
- 约束安全且主目标达标：step < 0 → lambda 缓慢衰减 → 给搜索更多探索空间

### 为什么是乘性而非加性？

不同约束的 lambda 量级差异巨大（1,000 到 80,000）。加性更新对小 lambda 修正过猛、对大 lambda 修正不足。乘性更新具有**尺度不变性**：同样的压力产生同样比例的调整，不管 lambda 当前是 1,000 还是 80,000。

### 内建的损失厌恶

一种典型配置会让收紧步长显著大于放松步长，例如：

```
delta_up   = +0.25  （收紧，约束被违反时）
delta_down = -0.08  （放松，约束被满足时）
```

收紧步长明显大于放松步长。这编码了一个根本性的业务不对称：指标不达标（例如订单跌破目标）对业务信任的损害，远大于在约束上略有冗余的代价。系统被刻意设计为损失厌恶型。

## 正交性：为什么这种分解是好的

### 几何独立性

从搜索引擎的视角：

- **信念通道**调整 `target_range`——约束边界的**位置**。它平移了优化景观中的可行域。
- **偏好通道**调整 `penalty_weight`——约束边界的**硬度**。它改变越界的代价，但不移动边界本身。

一个作用于目标函数，另一个作用于约束面。在优化问题的几何空间中，两者天然正交——调一个不会干扰另一个的修正效果。

### 互补的更新节律

| | 信念通道 | 偏好通道 |
|---|---|---|
| 更新速度 | 慢（eta=0.2，小步） | 快（压力驱动，违约时立即响应） |
| 持久性 | 写入 Memory DB，永久积累 | 写入配置，每轮可覆盖 |
| 防冲突 | 无冻结 | 信念大幅更新后冻结 1 轮 |
| 性格 | 学习（认知沉淀） | 反射（应激反应） |

冻结机制的原因：如果两个通道同时做大幅调整，下一轮的结果无法区分哪个调整起了作用、哪个调过了头。冻结 1 轮留出观察窗口，保持**可归因性**。

## 修正决策树

当线上结果不佳时，系统遵循以下诊断逻辑：

```
线上指标表现不佳
    |
    +-- 离线预测准确吗？
    |       |
    |       +-- 不准 --> 认知性错误：调截距（不碰惩罚）
    |       |
    |       +-- 准确 --> 惩罚权重够大吗？
    |                       |
    |                       +-- 不够 --> 价值性错误：加大惩罚（不碰截距）
    |                       |
    |                       +-- 够了 --> 约束规格错误：升级到第一层（人工修订）
```

这不仅是调试工具——它就是系统每轮实际执行的逻辑。信念通道修正映射误差，偏好通道修正惩罚误差，两者永不交叉污染。
