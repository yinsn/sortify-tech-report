# 影响力份额：一种可分解的排序度量

## 现有指标的问题

在多因子排序系统中，最终分数是各项之和：

```
S_i = order_i + gmv_i + ecpm_i + other_i
```

根本问题是：**当系统决定商品 A 排在商品 B 前面时，这个决策有多少是由每个因子驱动的？**

最常用的工具——**Kendall Tau**——回答不了这个问题。它能衡量两个排序之间的相关性，但对于搜参场景有四个致命缺陷：

| 缺陷 | 为什么致命 |
|------|-----------|
| **不可分解** | Kendall Tau 只能告诉你"GMV 与最终排序的相关性是 0.7"，但不能告诉你"GMV 贡献了 40%，订单贡献了 35%，广告贡献了 18%"。各因子的 tau 值相互独立，可以同时都很高，加起来也不等于任何有意义的东西。 |
| **对贡献大小视而不见** | 每对商品只有"一致/不一致"的二值判断。一对商品 GMV 差异 10000 和差异 1 的，Kendall Tau 一视同仁——但前者的排序几乎完全由 GMV 决定。 |
| **混淆直接因果与间接相关** | 如果 GMV 和订单高度相关，即使把 GMV 权重设为零，tau(GMV, 最终排名) 依然很高，因为订单间接抬升了高 GMV 商品的排名。Kendall Tau 分不清"GMV 在驱动排序"和"GMV 恰好跟驱动排序的因子相关"。 |
| **与参数非单调** | 增大某因子的权重，tau 有时上升有时下降。这使得 tau 无法作为优化目标。 |

Sortify 需要一个可分解为互斥百分比、对贡献大小敏感、不受间接相关污染、并能为搜索提供更直接方向性信号的指标。影响力份额满足前三点，并在实践中比 Kendall Tau 更适合作为优化反馈。

## 核心洞察：排序即逐对比较

排序不是一组绝对分数的序列——而是一组逐对决策的集合。A 排在 B 前面，是因为 S_A > S_B。真正起决定作用的是**分差**：

```
Delta_S(A,B) = Delta_order + Delta_gmv + Delta_ecpm + Delta_other
```

其中 Delta_order = order_A - order_B，以此类推。

分差揭示了博弈中的各股"力量"：每一项把 A 推向前方（正值）或拉向后方（负值）。排序结果由这些竞争力量的合力决定。

## 份额：量化每股力量的贡献

对于给定的商品对 (A, B)，每个因子的**份额**是其绝对贡献占总力量的比例：

```
share_order(A,B) = |Delta_order| / (|Delta_order| + |Delta_gmv| + |Delta_ecpm| + |Delta_other|)
share_gmv(A,B)   = |Delta_gmv|   / (|Delta_order| + |Delta_gmv| + |Delta_ecpm| + |Delta_other|)
...
```

为什么用绝对值？因为一个*对抗*排序结果的因子仍然在施加力量。如果 A 的订单分高 100 但 B 的 GMV 分高 100，两个因子各贡献了 50% 的决策力，尽管它们方向相反。

份额之和恒等于 1，构成完整的分配：**每个因子获得"排序决策蛋糕"的一定百分比。**

### 具体计算示例

```
商品 A (排名 1): order=120, gmv=300, ecpm=0,  other=5
商品 B (排名 2): order=100, gmv=350, ecpm=0,  other=3

Delta_order = +20    （推 A 向前）
Delta_gmv   = -50    （推 B 向前）
Delta_ecpm  = 0      （无影响）
Delta_other = +2     （略推 A 向前）

总力量 = |20| + |50| + |0| + |2| = 72

share_order = 20/72  = 27.8%
share_gmv   = 50/72  = 69.4%   <-- GMV 主导这次比较
share_ecpm  = 0/72   = 0.0%
share_other = 2/72   = 2.8%
```

解读：A 排在 B 前面，但这个结果主要由两者的 GMV 差异驱动（69.4%）。如果降低 GMV 的权重，B 很可能反超 A。

## 从局部到全局

单个商品对的份额是局部的。Sortify 先在每个 request 内对所有相关商品对加权聚合，再跨所有 request 聚合。

### 哪些商品对重要？

并非所有商品对都同等重要。第 50 名和第 51 名的排序对用户体验几乎没有影响——只有头部位置才真正重要。Sortify 选取 **Top K + Buffer** 的商品对（例如 Top 5 加上 15 的缓冲区，以捕获参数调整后可能进入头部的商品）。

### 位置衰减加权

头部商品对（第 1 名 vs 第 2 名）比尾部商品对（第 15 名 vs 第 20 名）更重要。Sortify 使用指数衰减权重：

```
weight(A,B) = exp(-min(rank_A, rank_B) / tau)
```

tau 控制衰减速度。tau=10 时，涉及第 1 名的商品对权重 0.90，第 5 名的 0.61，第 15 名的 0.22。指标自然聚焦于用户实际看到的位置。

全局**影响力**是所有商品对和所有 request 的加权份额平均：

```
I_gmv = SUM(weight * share_gmv) / SUM(weight)
```

关键性质：所有因子的影响力值之和仍然为 1——**总的排序决策力始终是 100%，分配为一块蛋糕。**

## 核心数学性质

**可加性**：I_order + I_gmv + I_ecpm + I_other = 1，恒成立。这使得权衡推理精确化："将 5% 的影响力从订单转移到 GMV"是一个定义明确的操作。

**方向性**：在单个商品对、其他项固定时，增大某因子的原始贡献会提高该因子的局部份额。对完整系统的聚合影响力而言，由于 Top-K 选对集合和排名本身也会变化，严格的全局单调性并不在所有配置下都成立；但它通常仍比 Kendall Tau 提供更直接的优化信号。

**因果隔离**：如果某因子的权重设为零，其影响力严格为零，不受其与其他因子相关性的影响。而 Kendall Tau 由于间接效应仍会显示高相关性。

## 因子不独立时怎么办

真实排序公式中的因子可能统计相关（高 GMV 商品往往订单量也高），也可能共享子表达式（订单和 GMV 使用相同的基础分乘数）。这会破坏分解吗？

**统计相关**：分解在数学上仍然有效。每对商品的"GMV 贡献了 X%"依然是精确陈述。但相关性会压缩*可交换空间*——如果两个因子近乎共线，就很难单独提升一个的影响力而不按比例影响另一个。这是数据本身的内在约束，不是方法的缺陷。

**共享子表达式**：某些参数会同时缩放多个因子，属于"全局旋钮"，无法在因子间重新分配影响力。只有主要作用于单一因子的参数族，才能实现更直接的因子交换。分解本身仍然成立；变化的是可交换空间的大小。

## "定量兑换"的含义

"影响力置换"指的是对 100% 排序决策蛋糕的受控重新分配。业务表述是：

> "当前 GMV 占排序决策的 40%，订单占 35%。我们希望把 GMV 提升到 45%，同时订单不低于 34%。"

影响力份额将此转化为精确的数学优化问题。搜索引擎找到一组参数，在约束条件下实现目标分配。没有可分解、可加的指标，这种定向权衡推理不可能实现。

## 一小组可调参数

在实际系统中，理论上的"调节权重"会落到一小组可调乘子上，大致覆盖以下几类控制：

| 参数类别 | 控制对象 |
|------|----------|
| 广告侧订单乘子 | 调整广告流量中的订单强调程度 |
| 广告侧 GMV 乘子 | 调整广告流量中的 GMV 强调程度 |
| 自然流量订单乘子 | 调整自然流量中的订单强调程度 |
| 自然流量 GMV 乘子 | 调整自然流量中的 GMV 强调程度 |
| 共享缩放乘子 | 同时影响多个自然排序子项 |
| 广告出价压力乘子 | 调整竞价项对排序的影响 |
| 价格敏感度乘子 | 调整价格相关项的非线性强度 |

将广告与自然流量拆开控制，可以在不显著扰动另一侧机制的前提下做更细的权衡。部分乘子属于全局旋钮，部分则更接近因子交换旋钮。

这些参数通常以 1.0 作为"不调整"基线，并在一个较宽的正数范围内搜索。
