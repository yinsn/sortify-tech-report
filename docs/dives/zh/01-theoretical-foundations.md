# 理论基础：信念与偏好为何必须分离

## 核心公理

Sortify 的架构受到 L.J. Savage（1954）的**主观期望效用理论（SEU）**启发：

> 在 Savage 的表示框架下，如果决策者的选择满足一组一致性公理，其偏好可以被表示为两类成分的组合：对世界状态的主观概率（信念），以及对结果的效用评估（偏好）。工程上可以把它理解为对“会发生什么”和“结果值不值得”的两类判断。

用公式表达：

```
a* = argmin_a  SUM_theta  P(theta | D)  x  L(a, theta)
                           -----------     -----------
                             信念            偏好
```

这不是纯粹的工程审美。SEU 在这里提供的关键启发是：**如果把对世界的判断和对结果的取舍压进同一个调节旋钮，系统会更难解释、诊断和校准。**

## 混淆之后会发生什么

在实践中，排序优化系统经常违反这一分离原则。当线上 GMV 下降时，典型的反应是"收紧约束"——一个笼统的调整，把两个根本不同的问题搅在一起：

1. **我们的预测错了吗？**（离线模型预测 GMV 会涨，但实际没涨）
2. **我们的容忍度错了吗？**（GMV 下降了一个我们认为可接受的幅度，但事实证明即便这个幅度也不可接受）

这两个问题需要截然相反的修正，却产生完全相同的症状。当它们被同一个机制处理时，两种认知偏差会浮现：

- **一厢情愿**：系统太渴望目标达成，于是扭曲了对离线-线上转化关系的客观认知，让预测看起来比实际更好。
- **酸葡萄心理**：系统的预测持续失败，于是降低对约束的重视程度——"反正我达不到，那这条约束大概也没那么重要。"

两种偏差都是自我强化的反馈环。一旦系统开始混淆"我认为会发生什么"和"我希望发生什么"，就没有内在机制能检测到这种腐化。

## Sortify 的操作化实现

Sortify 在架构层面强制执行 SEU 的分离原则：

| | 信念通道 | 偏好通道 |
|---|---|---|
| **Savage 对应** | P(theta\|D) — 关于世界状态的概率 | L(a, theta) — 关于结果的损失函数 |
| **系统提问** | "离线指标如何映射到线上现实？" | "约束被违反时有多痛？" |
| **控制对象** | `target_range` — 约束边界的位置 | `penalty_weight` — 越界的代价 |
| **修正的误差** | 认知性误差：世界模型不准 | 价值性误差：效用函数失校 |

两个通道不仅在概念上分离——它们作用于优化景观中**几何正交**的两个维度。信念在水平方向移动约束边界（改变"边界在哪里"）。偏好在垂直方向缩放违约代价（改变"越界有多痛"）。一个轴上的修正对另一个轴的效果为零。

## 两种正交错误，两条相反的修正路径

Sortify 设计中最深层的洞察是：智能体可以在两个正交方向上犯错，而混淆它们会导致严格更差的结果。

### 认知性错误："我以为 X 会发生，但并没有"

系统预测离线 I_order 提升 +10% 会带来线上订单正增长，结果订单下降了。世界的运行方式与模型的预期不符。

**正确应对**：更新信念。通过 LMS 回归和/或 LLM 截距跳变调整迁移函数。系统在承认："我对离线到线上的映射存在偏差，需要重新校准。"

**错误应对**：增大订单约束的惩罚权重。这确实让搜索更保守了，但底层映射仍然是错的——系统只是在一个扭曲的坐标系里更用力地搜索。

### 价值性错误："发生了，但我没想到会这么痛"

系统准确预测了广告收入会轻微下降。下降确实发生了，幅度也符合预期。但这个微小的下降却触发了一连串业务告警——约束设得太松了。

**正确应对**：增大惩罚权重。映射是准确的；价值评估出了问题。系统在学习："这条约束比我以为的更硬，需要更严厉地对待违约。"

**错误应对**：调整截距。预测本身是准确的——移动映射关系只会让预测变得*更不准*。

## 双时间尺度：积累与反射

两个通道还在不同的时间尺度上运作，形成互补的节奏：

| | 信念通道 | 偏好通道 |
|---|---|---|
| **更新速度** | 慢（LMS：每轮 eta=0.2） | 快（乘性更新：单轮可显著收紧） |
| **持久性** | 写入 Memory DB，永久积累 | 写入配置，每轮可覆盖 |
| **性格** | 学习——渐进的认知沉淀 | 反射——对疼痛的即时反应 |

信念像校准科学仪器：每次测量都会微调校准值，且这种微调永久保留。偏好像调节痛觉灵敏度：碰到热炉子时反应是即时的且不对称的（再碰一次的惩罚增长速度远快于安全一段时间后惩罚的衰减速度）。

这种双时间尺度设计赋予系统**长期稳定性**（信念变化缓慢，单轮噪声不会造成剧烈摇摆）和**短期响应力**（偏好在约束被违反时立即反应，不会反复犯同样的错误）。

## 三层架构是这一设计取向的自然结果

信念-偏好分离加上"LLM 能做什么"与"数值搜索能做什么"的区分，共同推导出三层架构：

```
第一层（人工/配置）：  定义目标和约束  ——  "宪法"
       |
第二层（LLM + 算法）：校准信念和偏好  ——  "教练组"
       |
第三层（Optuna TPE）：在校准后的框架内寻找最优参数  ——  "运动员"
```

- **第一层**是静态的：人工定义优化什么（GMV 影响力）和保护什么（订单、广告收入），低频变动。
- **第二层**是自适应的：每轮自动校准约束景观。积累的经验存储于此。
- **第三层**是反应式的：给定约束景观，找当前数据集下的最优参数。其输出是一次性事实，不是可复用的知识。

每一层恰好处理它擅长的问题类型。没有哪一层试图替代另一层。

## 对实践者的意义

这种设计的实际后果是**可诊断性**。当某个线上指标表现不佳时：

1. 检查信念通道：离线-线上映射准确吗？如果不准，截距需要调整。
2. 检查偏好通道：惩罚权重够大吗？如果映射准确但约束仍被违反，惩罚需要加大。
3. 检查搜索：Optuna 在给定约束下找到好的解了吗？如果两个通道都校准正确但结果仍差，可能是约束规格本身（第一层）需要人工修订。

没有分离时，你面对的是一个黑箱："出了问题，但不知道是预测的问题、惩罚的问题、还是搜索的问题。"有了分离，每种失效模式都有对应的诊断手段和对应的修复路径。
