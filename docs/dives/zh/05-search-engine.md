# 搜索引擎：Optuna TPE 约束优化

> 前置阅读：[影响力份额](02-influence-share.md) 定义了搜索要优化的指标。

## 搜索引擎做什么

搜索引擎位于 Sortify 三层架构的第三层。给定固定的离线数据集和校准后的约束框架（第二层输出的 target_range 和 penalty_weight），它找到最大化主目标影响力、同时尊重所有业务约束的参数向量。

```
输入：离线数据集 + target_ranges + penalty_weights
输出：最优参数向量（若干个值）
方法：数千次试验、并行 worker、Optuna TPE
```

搜索是纯数值过程。它不推理离线-线上差异，不跨轮次学习。它的任务是对一个定义良好的优化问题找最优解。

## 目标函数

每次试验通过一个标量得分评估候选参数：

```
objective = 正向收益 - 惩罚总和
```

### 正向收益

```
positive_score = 10.0 × uplift(I_gmv)

uplift(I_gmv) = (I_gmv_new - I_gmv_base) / |I_gmv_base|
```

基线 `I_gmv_base` 用所有参数 = 1.0 计算。使用相对提升率而非绝对值，保证不同量纲的指标可比。

### 惩罚：二次约束

每条约束在违反时施加**二次惩罚**。公开文档里不固定披露生产环境的具体阈值和权重；关键是每条 guardrail 都会定义：

- 一个允许区间：描述"轻微波动可接受到什么程度"
- 一个惩罚权重：描述"越界后要付出多大代价"
- 一个业务含义：说明它保护的是展示稳定性、订单、收入或其他关键指标

惩罚公式：

```
若 uplift 在允许范围内 → penalty = 0
若 uplift 超出范围 → violation = 超出距离 → penalty = weight × violation²
```

### 为什么用二次惩罚？

1. **边界附近宽容，远离边界严惩**：违反 0.1% 的惩罚是 `weight × 0.001²`，违反 10% 是 `weight × 0.1²`——后者是前者的 10,000 倍。搜索倾向于找"多个约束都轻微超出"而非"一个约束严重违规"的解。

2. **连续的目标塑形**：相较于硬阈值淘汰，二次惩罚会随着越界程度连续增加，为采样器提供更丰富的分数差异；靠近边界和严重违规的候选解因此能被更清楚地区分。

### 具体数值示例

基线：I_gmv_base = 0.40，I_order_base = 0.35

**好的试验**（参数 A）：
```
I_gmv = 0.44 → uplift = +10% → positive = 10 × 0.10 = 1.0
I_order = 0.348 → uplift = -0.57% → 在 [-1%, +100%] 内 → penalty = 0
目标分 = 1.0 ✓
```

**差的试验**（参数 B）：
```
I_gmv = 0.46 → uplift = +15% → positive = 10 × 0.15 = 1.5
I_order = 0.30 → uplift = -14.3% → 超出 -1% 达 13.3%
penalty = 10000 × 0.133² = 176.89
目标分 = 1.5 - 176.89 = -175.39 ✗
```

参数 B 的 GMV 提升更高（+15% vs +10%），但以订单影响力暴跌 14.3% 为代价。一个足够大的二次惩罚会彻底碾压这点正向收益。

## Optuna TPE 采样策略

### 为什么不用网格搜索？

如果对多个参数做均匀网格离散，评估次数会随维度指数爆炸。随机搜索虽然可行，但后期试验仍然像前期一样"盲目"。

### TPE 工作原理

1. **探索阶段**（前 N 次）：广泛采样，类似随机搜索
2. **分割**：按得分排序，前 10% 归入"好"组，其余归入"差"组
3. **建模**：对两组分别用核密度估计拟合概率密度 l(x) 和 g(x)
4. **采样**：下一个试验点选在 l(x)/g(x) 最大处——"好试验中常见、差试验中少见"的区域
5. **迭代**：每次完成试验后更新密度模型

行为轨迹：**广泛探索 → 发现有前途的区域 → 逐步聚焦 → 精细搜索**。在中等维度问题上，数千次试验通常足以找到很强的候选解。

### 并行策略：Constant Liar

并行 worker 运行时，用 **Constant Liar** 策略避免重复探索同一区域：正在评估中的试验会被临时视作已返回一个中性分数，促使其他 worker 自然避开其附近。多个 worker 通过共享的 study 后端协调。

## 校准如何影响搜索

搜索引擎不知道也不在乎离线-线上差异。它在收到的约束框架内优化。关键在于第二层持续校准这个框架：

**没有校准**（第 1 轮）：target_range 基于初始猜测，假设了一个不成立的离线-线上关系。搜索找到的"最优"参数在线上表现不佳。

**有校准**（第 7 轮）：经过 6 轮 LMS 和 LLM 修正，target_range 会比初始猜测更贴近真实的离线-线上关系。搜索在一个更校准的空间中操作，其"最优解"也更有机会在线上表现良好。

搜索引擎的输出质量取决于框架的准确度。Sortify 的双通道校准目标，是让这个框架在连续轮次中逐步贴近现实。
